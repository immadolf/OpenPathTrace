OpenPathTrace 最小 PRD 与实现计划

目标是复刻 PathTrace 的核心价值，但只做一个可审计、自用、无网络、最小闭环的 macOS 文件弹窗助手。

1. 结论

第一版只做一件事：标准 macOS 文件弹窗出现时，在弹窗旁显示常用目录、最近跳转目录和 Finder 当前窗口目录，点击后让弹窗跳转到对应目录。

不做全量 PathTrace。拖拽暂存、统计、菜单栏收藏面板、第三方文件管理器、复杂分组、自动选择文件全部后置。

2. 目标与非目标

目标

  • 菜单栏常驻。
  • 监听标准 NSOpenPanel / NSSavePanel。
  • 弹窗出现后显示非抢焦点贴边面板。
  • 列表包含收藏、最近、Finder 三组路径。
  • 点击路径后只跳转目录，不自动选择文件。
  • 本进程收到 AX 事件后，显示面板耗时小于 50ms。
  • 配置本地 JSON 保存，可人工审计。
  • 支持登录启动开关。

非目标

  • 不做 App Store、签名、公证、DMG。
  • 不启用 App Sandbox。
  • 不做自动更新。
  • 不加第三方依赖。
  • 不联网。
  • 不扫描全盘路径。
  • 不支持 Electron 或自绘文件弹窗的承诺。
  • 不承诺系统事件到达本进程之前的延迟。

4. 用户流程

  1. 用户启动 OpenPathTrace，菜单栏出现状态图标。
  2. 首次运行时检查辅助功能权限；缺失则引导打开系统设置。
  3. 用户在任意 App 中打开标准“打开”或“保存”弹窗。
  4. 工具通过 Accessibility 事件识别当前文件弹窗。
  5. 工具在弹窗右侧显示非抢焦点 NSPanel；右侧空间不足则贴左侧。
  6. 用户点击某个目录。
  7. 工具对文件弹窗发送 Cmd+Shift+G，输入路径并回车。
  8. 跳转成功后，该目录进入最近列表。
  9. 文件弹窗关闭后，贴边面板隐藏。

5. 权限与安全边界

项目决定
辅助功能权限必须申请，用于监听和操作文件弹窗。
Finder 自动化权限首次读取 Finder 当前窗口路径时由系统弹窗授权；失败则隐藏 Finder 分组。
网络不写任何网络请求代码。
存储只写本地 JSON，路径可见、可删、可备份。
文件内容不读取文件内容，只保存目录路径字符串。
Sandbox不开启，避免阻碍 Accessibility 与自动化。

7. 模块设计

模块职责
AppDelegate启动应用、初始化菜单栏、权限检查、组件装配。
StatusMenuController菜单栏状态、退出、打开配置文件、登录启动开关。
AccessibilityMonitor监听当前 App 的窗口创建和焦点窗口变化，识别标准文件弹窗。
FileDialogDetector判断 AX 窗口是否像标准文件弹窗，返回窗口 frame 和目标引用。
OverlayPanelController创建非抢焦点 NSPanel，计算贴边位置，渲染路径列表。
PathStore读写 JSON，维护收藏和最近路径。
FinderPathProvider通过 AppleScript 只读获取 Finder 当前窗口目录。
DialogNavigator点击路径后发送 Cmd+Shift+G、路径、回车。
LaunchAtLoginController封装 SMAppService.mainApp.register() 和 unregister()。

8. UI 规则

  • 面板默认贴文件弹窗右侧；右侧不够贴左侧；左右都不够则覆盖在右上角。
  • 面板不抢焦点。
  • 分组只显示“收藏”“最近”“Finder”。
  • 每项显示文件夹名；完整路径放 tooltip。
  • 顶部搜索框只过滤当前已加载列表，大小写不敏感。
  • 右键菜单只支持收藏和移除收藏。

11. 风险

  • 文件弹窗识别可能不稳定。 先只支持标准弹窗，遇到常用 App 再加针对性规则。
  • 系统事件到达时间不可控。 只承诺本进程处理耗时，不承诺系统全链路小于 50ms。
  • AppleScript Finder 权限可能被拒绝。 Finder 分组降级隐藏，不影响收藏和最近路径。
  • Cmd+Shift+G 依赖系统行为。 这是第一版最稳、代码最少的跳转方式；后续有必要再研究 AX 直接设置路径。
